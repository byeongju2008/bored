# 🚀 프로젝트명: Bored?

## 📌 한 줄 소개

심심할 때 같이 할 사람을 빠르게 찾는 웹사이트

---

## 🎯 문제 정의

요즘 사람들은 할 것은 많지만 같이 할 사람을 찾기 어렵다.
디스코드는 친목 위주라 처음 들어가기 어렵고,
새로운 사람을 쉽게 구하는 구조가 아니다.

---

## 💡 해결 방법

사용자가 방을 만들어
같이 할 사람을 모집하고
바로 채팅 및 음성으로 연결되는 웹사이트를 만든다.

---

## 🔥 핵심 기능

### 1. 사람 모집 시스템

* 방 생성
* 방 목록
* 태그 기반 검색

### 2. 실시간 소통

* 채팅
* 음성 채팅

### 3. 방 관리 시스템

* 방장 권한 (킥, 밴, 음소거)
* 경고 시스템 (사용자 설정 가능)
* 규칙 설정

### 4. 사용자 편의 기능

* 대기방
* 프로필 상태 표시
* 간단한 UI

---

## ⚙️ 기술 스택

* Backend: Python, Flask
* Frontend: HTML, Tailwind CSS, JavaScript
* Database: SQLite

---

# 🛡️ 프로젝트 설명 (말하기 버전)

제가 만든 프로젝트는 "Bored?"라는 웹사이트입니다.

이 프로젝트를 만들게 된 이유는
요즘 사람들이 같이 게임하거나 공부할 사람을 찾기 어렵다고 느꼈기 때문입니다.

디스코드 같은 경우는 이미 친해진 사람들끼리 사용하는 구조라서
처음 들어가면 어색하고 참여하기 어려운 문제가 있다고 생각했습니다.

그래서 저는
"누구나 쉽게 방을 만들고, 바로 사람을 모을 수 있는 서비스"를 만들고 싶었습니다.

---

이 사이트의 핵심 기능은 크게 3가지입니다.

첫 번째는 방 생성 기능입니다.
사용자가 방 제목, 설명, 태그를 입력해서
자기가 하고 싶은 활동을 바로 시작할 수 있게 만들었습니다.

두 번째는 실시간 소통 기능입니다.
채팅과 음성 채팅을 통해서
디스코드 같은 외부 앱 없이 바로 대화할 수 있도록 했습니다.

세 번째는 방 관리 기능입니다.
방장이 직접 유저를 관리할 수 있도록
킥, 밴, 타임아웃 기능과
경고 시스템을 넣었습니다.

특히 경고 시스템은
방장이 원하는 경우에만 켤 수 있게 하고
경고 횟수도 직접 설정할 수 있도록 해서
자유도를 높였습니다.

---

기술 선택 이유는 다음과 같습니다.

백엔드는 웹사이트 뒤에서 돌아가는 부분으로, 사용자가 보지 못하는 서버 쪽 기능을 말합니다. 예를 들어 로그인 처리, 데이터 저장, 방 생성 같은 기능을 담당합니다. 저는 Python의 Flask를 사용했습니다.

데이터베이스는 SQLite를 사용했습니다.
설치가 필요 없고 빠르게 개발할 수 있기 때문입니다.

프론트엔드는 사용자가 실제로 보고, 클릭하고, 사용하는 화면 부분을 의미합니다. 즉 웹사이트에서 눈에 보이는 모든 요소들, 예를 들어 버튼, 글자, 색상, 레이아웃 같은 것들을 만드는 영역입니다. HTML, CSS, 그리고 Tailwind CSS를 사용해서
빠르게 UI를 만들 수 있도록 했습니다.\

---

이 프로젝트를 통해
웹사이트가 어떻게 동작하는지 전체 흐름을 이해하게 되었고
사용자 입장에서 기능을 설계하는 경험을 할 수 있었습니다.

---

한 줄로 정리하면
"심심한 사람들이 쉽게 모여서 같이 활동할 수 있도록 만든 웹 서비스입니다."
